iv. How the Loading State is managed.
Ans. 
        The loading state is managed by tracking the status of asynchronous operations (like data fetching) 
    with dedicated state variables, typically using a boolean flag or a more comprehensive enum that represents 
    the different states (idle, loading, success, error). The UI then conditionally renders a loading indicator 
    (spinner, progress bar, skeleton screen) while the operation is in progress, and the actual content or an error 
    message upon completion. 

    Boolean Flags: The simplest method involves a boolean variable, for example, isLoading, which is set to true before 
    an asynchronous operation starts and false in the finally block or after the operation completes 
    (whether successfully or with an error).

    Enums/Sealed Classes: A more robust approach uses an enum or sealed class with associated data to handle multiple 
    potential states cleanly:
        Idle / None: Initial or inactive state.
        Loading: The operation is in progress.
        Success(Data): The operation completed successfully and holds the result data.
        Error(ErrorDetails): An error occurred, often including details about the error.

vii. How to implement and use eslint.
Ans. 
        To implement ESLint in a JavaScript project, install it via npm (npm init @eslint/config@latest), answer 
    onfiguration prompts to create a config file, and install the ESLint extension in VS Code for real-time 
    feedback. Run it using npx eslint or script commands to identify and fix code style/quality issues. 

    Implementing ESLint (Step-by-Step)
        1.  Initialize Project: Ensure you have a package.json file. If not, run npm init -y.
        2.  Install ESLint: Run npm init @eslint/config@latest to download the latest version and create an
            eslint.config.js (or .eslintrc) file.
        3.  Configure: The CLI will ask about your project setup (JS modules, framework, TypeScript, browser/node) 
            to create the appropriate config.
        4.  Install Extension: Install the official ESLint extension in VS Code to see errors highlighted in real-time

    Using ESLint
        • Real-time Linting: With the VS Code extension, warnings and errors will appear as red/yellow underlines while you code.
        • Fixing Issues: Run npx eslint yourfile.js --fix to automatically fix auto-fixable issues.
        • Add Scripts: Add "lint": "eslint ." to your package.json scripts to run it easily via npm run lint.
        • Ignoring Files: Create an .eslintignore file to exclude files or folders (e.g., node_modules, dist)

ix. How to Preview Local Projects with Cloudflare Tunnel.
Ans. 
        To preview local projects using Cloudflare Tunnel, you need the local application running and the cloudflared
    client installed. The quickest method uses a single command to generate a temporary public URL, while a more 
    permanent setup involves creating a named tunnel and routing a custom domain through the Cloudflare Zero Trust 
    dashboard. 


    Quick Tunnel (Temporary URL)
    This is the easiest way to share a development site for quick testing or review without needing a custom domain. 

    • Run your local project on a specific port (e.g., http://localhost:3000) using its local development server.
    • Open your terminal or command prompt and ensure cloudflared is installed and in your system's PATH. You can install 
    it using package managers like Homebrew on macOS (brew install cloudflare/cloudflare/cloudflared).
    • Run the cloudflared command, specifying your local address and port: cloudflared tunnel --url http://localhost:3000
    • Access your project using the unique, temporary https://*.trycloudflare.com URL provided in the terminal output. 

x. What is PM2 and how is it used? 
Ans. 
        PM2 (Process Manager 2) is a production-grade daemon process manager for Node.js applications, designed to keep apps 
    running 24/7, enable automatic restarts upon crashes, and simplify application management. It facilitates zero-downtime 
    reloads, load balancing through cluster mode, and live monitoring of application logs and system metrics. 
   
    How PM2 is Used:
        • Installation: Installed globally via npm: npm install pm2 -g pm2 - NPM.
        • Start Applications: Start apps with pm2 start app.js --name "my-app".
        • Process Management: Use pm2 {Link: restart|reload|stop|delete} <app_name|id> to manage application states.
        • Monitoring: Use pm2 monit for a terminal-based dashboard or pm2 logs for real-time logging.
        • Cluster Mode: Scale applications to utilize all CPU cores with pm2 start app.js -i max.
        • Persistent Service: Use pm2 startup to generate scripts that automatically restart apps on server reboots.
        • Zero-Downtime Reload: pm2 reload all updates apps without dropping traffic.

xiii. What are web apps?
